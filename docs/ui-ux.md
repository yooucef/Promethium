# UI/UX Design Guidelines

## Visual Identity

The interface follows the **"Void/Neon"** theme, reflecting the deep subsurface (Void) and high-tech intelligence (Neon).

*   **Background**: Deep Navy (`#050B24`).
*   **Surface**: Lighter Navy (`#0A143A`).
*   **Primary Accent**: Electric Cyan (`#00F0FF`) - Used for active states, calls to action, and key highlights.
*   **Secondary Accent**: Teal (`#00BFA5`) - Used for success states and secondary information.
*   **Text**: White (`#FFFFFF`) for headers, Slate Grey (`#94A3B8`) for body text.

## Typography

*   **UI Elements**: Inter (Sans-serif). Clean, legible, professional.
*   **Data & Code**: JetBrains Mono (Monospace). Precise, technical.

## Layout Principles

*   **Sidebar Navigation**: Persistent access to primary modules (Dashboard, Datasets, Jobs).
*   **Card-Based Content**: Group related information into distinct cards with subtle borders.
*   **Dense Displays**: Seismic data requires high-density information. maximize screen real estate for visualizations.

## Accessibility

*   Ensure high contrast between text and background.
*   Use semantic HTML elements (buttons, inputs) to support screen readers.
*   Provide visual feedback (hover states, focus rings) for all interactive elements.
